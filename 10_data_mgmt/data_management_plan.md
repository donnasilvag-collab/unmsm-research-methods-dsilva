# Plan de gestión de datos

## Objetivo

Establecer criterios mínimos para recolectar, organizar, proteger, versionar y conservar los datos del estudio sobre gestión de riesgos de seguridad de la información en empresas peruanas de desarrollo de software.

## Tipos de datos previstos

- Datos cuantitativos de encuesta.
- Matrices de codificación y variables derivadas.
- Notas de entrevista, transcripciones autorizadas y memos analíticos.
- Documentos institucionales facilitados por las empresas participantes.

## Criterios de almacenamiento

- Los datos sensibles permanecerán fuera del repositorio público.
- Solo ingresarán al repositorio versiones anonimizadas, derivadas o punteros controlados por DVC.
- Cada conjunto deberá contar con nombre de versión, fecha y breve descripción de procedencia.

## Control de acceso

El acceso a datos identificables quedará restringido a la investigadora y, si corresponde, a personas expresamente autorizadas para supervisión académica.

## Conservación y cierre

Al finalizar el curso o la investigación, se conservarán únicamente los insumos necesarios para trazabilidad metodológica y respaldo académico, siempre bajo anonimización y resguardo ético.
